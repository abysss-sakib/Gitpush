### `app.py` — Flask Backend (1,254 lines)

The entire Python server. Responsibilities:

| Section | What it does |
|---|---|
| Config & init | Loads `.env`, sets up Flask, CORS, logging |
| Context management | Saves/loads active `{username, repo}` to `.gitpush_ctx.json` |
| `GET /` | Renders `index.html` template |
| `GET /health` | Returns build version + status JSON |
| `GET /repo-context` | Returns active username + repo |
| `POST /change-repo` | Sets active repository (validates with GitHub) |
| `POST /change-username` | Sets active GitHub user |
| `GET /token-check` | Validates GITHUB_TOKEN against GitHub API |
| `GET /repo-info` | Returns stars, forks, language, dates, etc. |
| `GET /git-status` | Returns last N commits |
| `GET /list-files` | Lists files/folders at a given path |
| `GET /list-tree` | Returns full recursive repo tree |
| `GET /read-file` | Returns file content (UTF-8 or base64) |
| `POST /upload-file` | Create or update a single file (JSON/base64) |
| `POST /upload-file-stream` | Upload file as multipart/form-data (memory-efficient) |
| `POST /create-blob-stream` | Upload a binary blob, returns SHA (used by folder upload) |
| `POST /commit-tree` | Commit N pre-uploaded blobs in ONE atomic commit |
| `POST /upload-folder-atomic` | Upload folder as one JSON batch (small files) |
| `POST /replace-file` | Replace existing file contents |
| `POST /delete-file` | Delete a single file |
| `POST /delete-many` | Delete N files/folders atomically |
| `POST /delete-all` | Wipe every file in the repo (keeps README) |

### `Procfile` — Gunicorn Configuration

```
web: gunicorn app:app --workers 1 --threads 8 --timeout 300 --bind 0.0.0.0:$PORT
```

| Flag | Value | Reason |
|---|---|---|
| `--workers 1` | 1 process | More workers cause "no repo selected" errors (each worker has separate memory) |
| `--threads 8` | 8 threads | Handles concurrent requests without multiple processes |
| `--timeout 300` | 5 minutes | Large folder uploads need time; default 30s is too short |
| `--bind 0.0.0.0:$PORT` | Dynamic port | Required by Render, Railway, Fly.io |

## Data Flow

```
Browser (app.js)
      │
      │  fetch("/api-route", { headers: X-Gitpush-Username, X-Gitpush-Repo })
      ▼
Flask (app.py)
      │
      │  PyGithub → GitHub REST API (api.github.com)
      ▼
GitHub (stores files, returns SHA, commit URLs)
      │
      ▼
Flask returns JSON { ok: true, ... }
      │
      ▼
Browser updates UI (panel, toast, hero stats)
```

The app is **stateless across server restarts** — active username and repo are sent by the browser on every request via HTTP headers (`X-Gitpush-Username`, `X-Gitpush-Repo`). A `.gitpush_ctx.json` file on disk serves as a fallback for fresh browser tabs.
